## Overview

Delete a leave type configuration. This is an administrative operation typically performed by HR managers. Leave types can only be deleted if they have no associated leave requests or balances.

<Warning>
  Deleting a leave type is permanent and cannot be undone. Ensure the leave type is not in use before deletion.
</Warning>

## Authentication

Requires authentication with Bearer token. Available to:
- SuperAdmin
- HR Manager

## Path parameters

<ParamField path="id" type="string" required>
  UUID of the leave type to delete
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the deletion was successful
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message: "leave type deleted"
</ResponseField>

<RequestExample>
```bash cURL
curl -X DELETE https://api.companyflow.com/leave-types/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer YOUR_TOKEN"
```


## Leave type management

Before deleting, you can manage leave types using these operations:

### Create leave type

`POST /companies/{company_id}/leave-types`

Create a new leave type with configuration for days allowed, paid status, documentation requirements, and carry-forward rules.

**Request body example:**

```json
{
  "name": "Annual Leave",
  "code": "AL",
  "description": "Standard annual vacation leave",
  "daysAllowed": 20,
  "isPaid": true,
  "requiresDocumentation": false,
  "carryForwardAllowed": true,
  "maxCarryForwardDays": 5,
  "colorCode": "#3498db",
  "status": "active"
}
```

### Update leave type

`PUT /leave-types/{id}` or `PATCH /leave-types/{id}`

Modify an existing leave type's configuration.

**Request body example:**

```json
{
  "daysAllowed": 22,
  "maxCarryForwardDays": 7,
  "status": "active"
}
```

### List leave types

`GET /companies/{company_id}/leave-types`

Retrieve all leave types for a company with filtering and pagination.

**Query parameters:**
- `page` - Page number (default: 1)
- `pageSize` - Items per page (default: 10)
- `status` - Filter by status (active/inactive)
- `search` - Search by name or code

### Get leave type

`GET /leave-types/{id}`

Retrieve details of a specific leave type including all configuration settings.

## Deletion constraints

The system enforces these rules when deleting leave types:

<AccordionGroup>
  <Accordion title="No active leave requests">
    Cannot delete if employees have pending or approved leave requests using this leave type. All requests must be completed, rejected, or withdrawn.
  </Accordion>
  
  <Accordion title="No leave balances">
    Cannot delete if employees have leave balances (used or available) for this leave type. Balances must be transferred or zeroed out first.
  </Accordion>
  
  <Accordion title="Company ownership">
    Leave types can only be deleted by administrators of the company that owns them.
  </Accordion>
</AccordionGroup>

## Alternative: Deactivate instead

<Tip>
  Instead of deleting, consider deactivating the leave type by setting its status to `inactive`. This:
  
  - Preserves historical data and reports
  - Prevents new leave requests using this type
  - Allows reactivation if needed
  - Maintains data integrity for past requests
</Tip>

**Deactivate a leave type:**

```bash
curl -X PATCH https://api.companyflow.com/leave-types/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "inactive"}'
```
